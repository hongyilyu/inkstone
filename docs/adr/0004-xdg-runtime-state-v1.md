# 0004 — XDG-only runtime state in v1

Inkstone v1 keeps everything it owns under XDG paths — user preferences and OAuth credentials in `~/.config/inkstone/` (`config.json`, `auth.json` mode 0600), session database and WAL in `~/.local/state/inkstone/` (`inkstone.db` + `-wal`/`-shm`) — with `vaultDir` declared by the user in `config.json` and the vault itself treated as user knowledge content only. Two natural extensions are deliberately deferred for v1 simplicity, not rejected on architectural grounds: vault-stored agent definitions (agents are code-defined in `src/backend/agent/agents/<name>/` today) and cross-machine session sharing scoped to a vault that may live at different paths per machine (sessions are machine-local today). When cross-machine sharing lands, `vaultDir` likely has to move out of `config.json` into a per-session DB column so a shared sessions DB can resolve paths against each machine's local vault — that's a constraint to revisit then, not a v1 decision.
