# 0006 — Action-first UX, persona-first context isolation

Inkstone's primary mental model is action-first: the user opens the program with a verb in mind ("read this article", "lint the knowledge base", "capture a daily note") and the slash-command registry is globally addressable from the open page so the user can invoke any agent's verbs without first picking the agent. Under the hood the architecture is persona-first for context isolation — picking a verb (or pressing Tab, or sending the first freeform message) **commits** the session to one agent for the rest of its in-memory lifetime, so a Reader session's article-workflow context can't bleed into a KB query and turn it into "let's read this query like an article." After commitment the slash autocomplete filters to the committed agent's verbs only (Tab + slash-pick + first-message all collapse the verb space identically); before commitment, the open-page autocomplete shows every agent's verbs. The two halves of the model coexist cleanly because "agent" is implementation vocabulary for the slash path (the registry resolves `/article → Reader` invisibly) and only surfaces as user-visible vocabulary in three places: the bubble footer (provenance, "▣ Reader · model · duration"), the router's dispatch bubble (cf. ADR 0007), and the explicit Tab override on the open page (which exists for power users who want to bypass the router and pre-commit before typing).
