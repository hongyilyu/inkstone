# 0003 — Folder-per-agent layout

Each custom agent lives in `src/backend/agent/agents/<name>/` as a self-contained unit (its own `index.ts`, instructions, state-coupled tools), so a contributor finds everything about an agent in one place and `rm -rf agents/<name>/` is a clean delete; adding the next agent (Researcher and beyond) is dropping a folder plus one registry line in `agents.ts`. Tool placement follows a categorical rule: agent-neutral tools live in the shared pool at `backend/agent/tools.ts` and are imported into an agent's `extraTools`, while state-coupled tools (none today, but the rule is in place pre-emptively) live under `agents/<name>/tools/` with any public helpers re-exported from the agent's `index.ts`. Biome enforces the boundary so the shell can't deep-import `agents/*/tools/*` and the TUI can't deep-import `@backend/agent/agents/**` — without that enforcement, the folders would drift back into a flat shell-touches-everything layout within a quarter.
