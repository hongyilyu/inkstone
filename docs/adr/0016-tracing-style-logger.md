# 0016 — Tracing-style logger built in-tree

Inkstone runs as a single-process TUI with an in-process pi-agent-core agent and a SQLite-backed persistence layer. Failures previously surfaced as bare `console.{error,warn}` calls hand-prefixed with strings like `[inkstone]` or `[routing-seam]` (20 sites across 12 files), with no level control, no file output, no structured fields, and no way to correlate events across an async turn boundary. Inside the OpenTUI alt-screen those stderr writes are also swept into OpenTUI's `Capture`, so the trail vanishes after a session ends.

The replacement is modeled on Rust's `tracing` crate: child loggers for namespaces, structured fields on every event, spans that bracket an async region with `enter`/`exit ok|err` events plus duration, and `AsyncLocalStorage`-based ambient context so events emitted deep inside a span auto-inherit its fields without parameter threading. A single human-readable line per event, sortable by leading ISO-8601 timestamp, written to `~/.local/state/inkstone/logs/inkstone.log` (path overridable via `INKSTONE_LOG_FILE`). Threshold defaults to `warn` in production and `debug` under `bun run dev` (the script sets `INKSTONE_LOG=debug`); user-overridable via `INKSTONE_LOG`.

The logger is built in-tree (~200 LOC, zero deps) rather than pulling in OpenTelemetry or pino. OpenTelemetry's footprint is wrong for a single-process TUI — it's designed for distributed tracing with OTLP exporters, semantic conventions, and resource detection, of which we use none. `pino` is fast structured logging but ships no span or `AsyncLocalStorage` primitives; we'd own the span/ALS layer regardless, and the formatter savings (~50 LOC) don't justify ~5 transitive deps and another upgrade surface. Bun's `node:async_hooks` works natively with no shim, so the in-tree implementation has no platform escape hatch.

The logger lives in `src/backend/logger/` (backend-owned, frontend-agnostic per the layer rules in `biome.json`'s `noRestrictedImports` overrides) — TUI imports via `@backend/logger`. The file sink is wired once from `src/index.tsx` via `initLogger()`; tests inject a memory sink via `setSink(...)` for hermeticity (Bun's test runner does not auto-set `NODE_ENV=test`, so env-detection would be brittle). `reportPersistenceError` (`backend/persistence/errors.ts`) keeps its `console.error` fallback for the pre-toast startup window and additionally routes through the logger so the file always carries the trail — toast-or-not.

`AsyncLocalStorage` propagates through `await`/`Promise.then`/`setTimeout`/`setImmediate` natively but breaks across event-emitter boundaries (the listener runs in the emitter's async context, not the registration's). The `logger.bind(fn)` helper snapshots the active context at registration and re-establishes it inside the callback. Two boundaries inside v1 spans are wrapped: `Session.subscribe` / `tool_execution_*` reducer handlers, and the `setTimeout(0)` continuation in routing-fork resume. Solid lifecycle hooks and OpenTUI input handlers are not wrapped today — they live outside any v1 span; wrap when an observed log line drops its parent fields.

Format includes a leading ISO-8601 UTC timestamp so `tail -f`'d output sorts naturally and a multi-session log file remains readable when multiple sessions interleave (every line carries `sessionId` as a structured field once the span boundaries land). Span nesting uses `>` in the namespace (`[agent.turn>tool]`) rather than dot-collapse so the chain is visually distinguishable from a single-namespace name.
