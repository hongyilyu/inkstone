# 0001 — Three-layer split: tui / bridge / backend

Inkstone splits `src/` into `backend/` (headless agent — pi-agent-core, providers, persistence), `bridge/` (pure type contract, zero runtime — `DisplayMessage`, `AgentStoreState`), and `tui/` (Solid + OpenTUI), with imports flowing one direction: tui → bridge, tui → backend, backend → bridge. `bridge/` exists because pi-agent-core and OpenTUI are independent upstreams whose shapes must reconcile in one place AND a web frontend is on the roadmap, making a second consumer of the view-state contract real rather than speculative; only types live in `bridge/` today, with runtime concerns like the event-reducer staying inline in `tui/` until the second frontend lands. Biome's `noRestrictedImports` requires `@tui/*`-style aliases for any cross-layer reference and also blocks relative-path equivalents (`../tui/...`), so the boundary can't be silently bypassed.
